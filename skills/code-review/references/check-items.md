# 审查项

按优先级从高到低排列，数字越小优先级越高，审查时优先检查高优先级项。

1. **安全漏洞**
   - XSS、SQL 注入、CSRF、路径穿越、权限绕过、反序列化漏洞
2. **硬编码凭据**
   - Token、密码、API Key
3. **逻辑错误**
   - 条件判断遗漏、竞态条件、状态不一致、循环边界错误
4. **并发/线程安全**
   - 缺少同步、使用非线程安全集合、潜在死锁、数据不一致
5. **边界条件**
   - 空指针、极值、特殊字符处理、事务缺失、缓存未失效、状态回滚遗漏
6. **错误处理与资源管理**
   - 异常未捕获、吞噬异常/错误、空异常处理块、错误码使用不当
   - 资源未关闭（如文件句柄、连接）、输入未校验
7. **破坏兼容性**
   - 接口变更未向后兼容、序列化格式变更（如 JSON 字段重命名）
   - 数据库 schema 变更未做迁移、配置项/环境变量变更、第三方 API 版本变更
8. **配置与环境**
   - 硬编码配置值、环境依赖未隔离
9. **性能问题**
   - 循环低效、N+1 查询
10. **日志/可观测性**
    - 关键路径缺日志、日志泄露敏感信息
11. **测试覆盖**
    - 新功能缺少测试、边界测试缺失
12. **残留代码**
    - 调试输出、TODO、注释掉的代码
    - 已废弃但未删除的方法/函数、调试用的特殊路由/端点
13. **文档缺失**
    - 改代码但未更新注释/文档
14. **代码复用**
    - 重复代码未抽取、过度抽象
